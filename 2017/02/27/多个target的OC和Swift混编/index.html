#OC和Swift混编

1. appDelegate从OC转为swift

在OC中，程序的入口是在main.m中，但是在swift中，你应该找不到这个东西，那么这个入口到底是在哪里呢

如果你仔细看的话，就在AppDelegate.swift中

其他的配置就不需要改变了

2. OC和swift混编

  • 1.在OC和swift的混编中，会自动创建桥接文件（你也可以选择不创建，桥接文件用于swift调用OC代码）

    n个target就会产生n个桥接文件，比如说契约所一个线上版本，一个测试版本，一个线上的widget，一个测试的widget，就会产生对应的四个桥接文件(其实只要两个桥接文件即可，下面会讲)

当然你也可以在Building Settings中自己设置桥接头文件（一般情况下我们会用系统默认生成的），如图1-3所示：

2.1 OC中调用Swift

配置Build Setting

修改Module

Swift的代码默认是使用module管理的，如果用xib布局的话，swift要是不加载，就会用到这个东西，还有Framework的导入等问题

配置OC中导入Swift的头文件和桥接文件

因为n个target对应n个桥接文件，所以最好就是都共用同一个桥接文件，将不用的桥接文件删除。在 Build Setting 中 设置 SWIFT_OBJC_INTERFACE_HEADER_NAME，将原来的$(SWIFT_MODULE_NAME)-Swift.h 替换成 $(PROJECT_NAME)-Swift.h（多个target中也是这么配置）

xxx-Swift.h就是在OC代码中需要导入的swift头文件

此处测试版的 Bridging Header和 Header Name都线上版本都是一样的。这样原先生成的beta版本的桥接文件就可以删除
